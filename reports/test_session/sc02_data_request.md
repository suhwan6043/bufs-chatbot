# 요청 1 — sc02 (장학금 최소 이수학점)

**유형**: 데이터 소스 간 숫자 불일치 (공식 PDF vs 학생포털 페이지)

---

## 1. 요약

부산외대 학사지원팀이 관리하는 **두 개의 공식 자료**가 "장학금 수혜에 필요한 최소 이수학점" 수치에서 상이한 값을 제시하고 있습니다. 공식 최신 PDF는 **12학점**을 기준으로 명시하고 있으나, 학생포털의 `수강신청안내` 페이지는 여전히 **15학점**으로 표기되어 있습니다.

같은 문장 구조에서 숫자만 다르므로 **학생포털 페이지가 구버전(갱신 누락)**일 가능성이 높습니다. 최신 PDF 기준으로 학생포털 페이지 갱신을 요청드립니다.

---

## 2. 두 소스 원문 비교

### 2-1. 공식 PDF (**12학점** — 최신본 추정)

- **파일**: `2026학년도 1학기 학사안내.pdf`
- **페이지**: p.8, "6. 수강신청 학점" 섹션, "나. 최소 신청 학점" 항목의 주석
- **원문**:

> ＊단, 장학금은 **12학점**(4학년은 9학점) 이상 취득자에 한해 지급함.

(주변 문맥: 학년별 최대/최소 신청 학점표 바로 아래의 보조 주석)

### 2-2. 학생포털 페이지 (**15학점** — 구버전 의심)

- **URL**: `https://m.bufs.ac.kr/Information/LESN6020.ASPX?mc=0966`
- **페이지명**: 학생포털시스템 → "수강신청안내"
- **원문**:

> 단, 장학금은 **15학점**(4학년은 9학점) 이상 취득자에 한해 지급함.

(동일한 문장 구조·동일한 4학년 예외(9학점)를 공유. 숫자만 12 → 15로 상이)

### 2-3. 참고: 상호 일치하는 장학 페이지 (12학점)

- **URL**: `https://m.bufs.ac.kr/Information/SAJH1020.ASPX?mc=0935` (학생장학시스템 교내장학금 페이지)
- **원문 일부**: "직전학기에 **12학점**(4학년은 9학점, 해외파견 학생은 10학점) 이상을 이수하고…"

→ 학사지원팀이 운영하는 장학 전용 페이지는 이미 **12학점**으로 맞춰져 있으며, 수강신청안내 페이지만 **15학점**으로 남아있는 불일치 상태입니다.

---

## 3. 요청 사항

1. **주(主) 요청**: `LESN6020.ASPX?mc=0966` (수강신청안내) 페이지의 해당 문장에서 "**15학점**" → "**12학점**"으로 수정.
   - 수정 대상 문장: `단, 장학금은 15학점(4학년은 9학점) 이상 취득자에 한해 지급함.`
   - 수정 후 예시: `단, 장학금은 12학점(4학년은 9학점) 이상 취득자에 한해 지급함.`
2. **부가 확인**: 학사지원팀 내부적으로 실제 기준이 12학점이 맞는지 재확인. (만약 기준이 15학점이고 PDF가 틀린 경우라면 오히려 PDF 다음 인쇄본 정정이 필요하므로 회신 요망.)
3. **선택 사항**: 학생포털 내 "장학금/수강신청" 관련 모든 페이지에 같은 문구가 다른 숫자로 산재해있는지 일괄 점검 (1회 스캔 권장).

---

## 4. 영향 범위

- **직접 영향**: 챗봇 평가셋(sc02) 1건 오답.
  - 질문 예: "장학금을 받으려면 한 학기에 몇 학점 이상 이수해야 하나요?"
  - 기대 정답: 12학점(4학년은 9학점)
  - 현재 챗봇 응답: 15학점 (학생포털 구버전 청크가 retrieval 상위에 위치한 결과)
- **학생 혼선 가능성**: 학생이 포털에서 수강신청 안내를 읽을 때 12학점과 15학점 사이에서 혼선 발생 가능. 4학년 학생(9학점 예외)은 영향 없음.

---

## 5. 챗봇 시스템 측 후속 조치 (참고용, 학사지원팀 회신과 별개)

학사지원팀 회신이 지연되거나 학생포털 갱신 전 임시 대응이 필요한 경우, 챗봇 리랭커에 다음 정책을 추가 검토 예정입니다(이번 요청의 필수 사항은 아님):

- **Tier 1 내 하위 우선순위**: 동일 주제에 대해 공식 PDF 청크와 학생포털 스크랩 청크가 경합할 경우, 수치가 다르면 PDF를 우선 인용.
- 해당 정책은 `app/pipeline/reranker.py`에서 메타데이터 기반으로 구현 가능하며, 소스 정리가 선행되면 정책 자체가 불필요해집니다.

## 6. 회신 요청

- **희망 처리 기한**: 가능하면 다음 학기 수강신청 안내 기간 전까지 (학생 혼선 최소화 목적).
- **문의**: 본 챗봇 프로젝트 담당 (개발 측)

## 7. 부록: 재현 방법 (개발 측 참고)

ChromaDB에서 두 소스가 같은 주제·다른 숫자로 동시에 색인된 것을 확인:

```python
from app.vectordb.chroma_store import ChromaStore
cs = ChromaStore()
coll = cs.collection

# 학생포털 15학점 청크
res1 = coll.get(
    where={"source_url": "https://m.bufs.ac.kr/Information/LESN6020.ASPX?mc=0966"},
    limit=20, include=["documents", "metadatas"]
)
# → "단, 장학금은 15학점(4학년은 9학점) 이상 취득자에 한해 지급함." 출현

# PDF 12학점 페이지
# data/pdfs/2026학년도1학기학사안내.pdf p.8에 "12학점(4학년은 9학점)" 출현
```

학생포털 페이지가 갱신되면 ChromaDB 해당 소스에 대해 재인제스트만 수행하면 됩니다 (`scripts/ingest_all.py` 또는 학생포털 전용 인제스트 스크립트).

# 요청 2 — s04 (2025학년도 전기 학위수여식 일정 부재)

**유형**: 데이터 부재 (ChromaDB 및 인제스트된 PDF에 정보 없음)

## 1. 요약

챗봇 평가셋 **s04** 문항: *"2025학년도 전기 학위수여식은 언제인가?"* (기대 GT: 2026년 2월 20일). 현재 챗봇은 "해당 일정 정보를 찾을 수 없습니다. 학사지원팀(051-509-5182)에 문의하시기 바랍니다"로 거부 응답을 반환합니다 (환각 없이 정상 거부).

조사 결과, 이 정보가 **ChromaDB 어떤 청크에도 존재하지 않으며**, 현재 인제스트된 `2026학년도1학기학사안내.pdf`의 학사일정표(p.5)에는 2025학년도 **후기** 학위수여식(8월 14일)만 있고 **전기** 학위수여식(2026년 2월) 정보가 없습니다.

## 2. 근거

### 2-1. ChromaDB 재현 (검색 실패)

```python
from app.vectordb.chroma_store import ChromaStore
cs = ChromaStore()
# 여러 쿼리 변형으로 검색 — 모두 "2월 20" 또는 "전기 학위수여" 포함 청크 0건
for q in ["2025학년도 전기 학위수여식", "2026년 2월 20일 학위수여",
          "전기 학위수여식 2026", "20260220 학위"]:
    res = cs.search(query=q, n_results=15)
    # → 관련 청크 미검출 (top-15 어디에도 2월 20일 또는 전기 학위수여식 없음)
```

### 2-2. PDF 원본 확인

`data/pdfs/2026학년도1학기학사안내.pdf` p.5 학사일정표의 학위수여식 항목:

> 8월 14일(금) 2025학년도 **후기** 학위수여식

→ **전기** 학위수여식 정보는 p.5 어디에도 없음 (1-8월 일정표이므로 이전 학년도 2월 일정이 포함되지 않는 구조).

## 3. 요청 사항

다음 중 하나 이상의 방법으로 **2025학년도 전기 학위수여식 일정(예상: 2026년 2월 20일 전후)** 데이터를 제공해 주시기 바랍니다.

1. **학사일정 공지 문서 제공**: "2025학년도 2학기 학사안내" 또는 "2025학년도 후학기 학사일정" PDF (현재 ChromaDB에 없음). 있다면 경로 전달.
2. **학위수여식 공지 URL**: 학교 홈페이지 학사공지에 게시된 학위수여식 안내 공지가 있다면 해당 URL (또는 `bufs.ac.kr/bbs/board.php?...` 링크).
3. **구조화된 일정 정보 직접 제공**: 2025학년도 전기(및 타 학기) 학위수여식 날짜·시간·장소를 간단히 회신.

## 4. 영향 범위

- 챗봇 평가셋 s04 1건 오답(refusal).
- 유사 패턴 (향후 "2024학년도 후기", "2026학년도 전기" 등) 문항도 동일 원인으로 실패할 가능성.

## 5. 챗봇 시스템 측 후속 조치 (참고용)

- 데이터 공급 시 `scripts/ingest_all.py` 해당 소스 추가 → 자동 인제스트.
- 이미 `query_analyzer.py`에 `semester_half` (전기/후기) 엔티티 추출이 구현되어 있으므로 데이터 추가 시 바로 retrieval/aligns 경로에서 활용됩니다.
