## 3. 검색 파이프라인 (하이브리드)

```
질의 → [0] follow-up 감지 + 쿼리 재작성 (옵션, gemma3:4b)
       [1] Phase 1: Dense 검색 (ChromaDB query, doc_type 우선순위)
       [2] Phase 2: 결과 부족 시 doc_type 미지정 확장
       [2.5] FAQ 최소 보장 (FAQ 카테고리에서 부족 시 추가)
       [3] Phase 3: BM25 sparse 후보 합류 (top 20)
       [4] Reranker (BGE Cross-Encoder, top_k=10)
       [5] Context Merger: 그래프 직접답변 + 벡터 결과 RRF 병합
       [6] LLM 답변 생성 (qwen3:8b)
       [7] Validator: refusal 감지, 환각 가드
```

| 파라미터 | 값 |
|---|---|
| dense top n_results | 15 |
| BM25 top n_results | 20 |
| reranker top_k | 10 |
| 검색 우선순위 (doc_type Tier) | Tier 1 (domestic, guide) > Tier 2 (faq, pinned notice) > Tier 3 (기타) |

## 6. 알려진 이슈 / 한계

| 이슈 | 영향 | 현황 |
|---|---|---|
| 신입생 가이드북(스캔 PDF) 미인덱싱 | 입학·교육과정 안내 답변 불가 | v2 파이프라인에서 OCR로 해결 (Surya 0.17) |
| 시간표 표 셀 단위 분절 | 시간표 의미 단위 검색 안 됨 | v2에서 마크다운 표 청크로 해결 (qwen2.5vl) |
| 학사안내 표 메타데이터 부재 | 섹션 헤더 검색 안 됨 | v2 `section_path` 메타 100% 채움 |
| `notice_attachment` 71% 차지 | 코퍼스 균형 부족 | 별도 lifecycle 메타로 정리 (v2) |
| PUA·OCR 마크업 글리프 잔재 | 검색 노이즈 | v2 `replace_pua` + `strip_ocr_markup`으로 정리 |
| query_rewriter `gemma3:4b` CPU fallback (팀원 환경) | 일부 환경 응답 지연 | `CONV_REWRITE_ENABLED=false` 또는 별도 머신 분리 검토 |
