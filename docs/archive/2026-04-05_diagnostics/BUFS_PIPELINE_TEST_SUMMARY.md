## 문제별 분류 (4가지 주요 이슈)

### 1. 신뢰도 계산 로직 완전 파괴 (CRITICAL)
**영향: 16/20 쿼리 (80%)**

**위치**: `app/pipeline/context_merger.py` 299-314줄

**문제**:
```python
# 현재 로직
if direct_answer:
    confidence = 1.0
else:
    # 이 리스트는 거의 항상 비어있음 (그래프 핸들러는 1.0 이상 반환)
    quality_scores = [r.score for r in graph_results if r.score < 1.0]
    if quality_scores:  # ← 거의 실행 안 됨
        confidence = max(quality_scores)
    elif _pre_rrf_vector_top > 0:  # ← 벡터 점수가 0.0인 경우가 대부분
        confidence = min(max(_pre_rrf_vector_top, 0.0), 1.0)
```

**증상 예시**:
- 쿼리: "수강신청 어떻게 해?"
- 벡터 결과: 5개, 그래프 결과: 5개 (충분한 컨텍스트!)
- 신뢰도: 0.10 (매우 낮음!)
- 원인: direct_answer 없음 → 폴백 → 불완전한 점수 계산

**해결책**:
```python
if direct_answer:
    confidence = 1.0
elif len(selected_vector) + len(selected_graph) >= 3:
    confidence = 0.7  # "충분한 컨텍스트 있음, LLM이 합성하면 좋은 답변"
elif len(selected_vector) + len(selected_graph) > 0:
    confidence = 0.5  # "최소 컨텍스트 있음"
else:
    confidence = 0.0  # "컨텍스트 없음"
```

**기대 효과**: 16개 쿼리의 신뢰도가 0.06-0.18 → 0.5-0.7로 상향
