## 📌 정정사항 (2026-04-26)

### A. 깨짐 비율 측정 정규식 버그 발견·수정

기존 측정에 사용한 정규식 `[㐀-䶿--   Ȁ-ʯ]`에서 **dash가 range로 잘못 해석**되어 정상 ASCII 문자(0-9, `|`, `.`, `;` 등)가 깨짐으로 카운트됐음. 결과적으로 모든 깨짐 비율이 부풀려져 측정됨.

**정정 후 실측**:

| 지표 | 이전 측정 (잘못됨) | 정정 측정 (정확) |
|------|-------------------|-----------------|
| 전체 깨짐 글자 비율 | (측정 시 ASCII 포함) | **0.89%** |
| 깨짐 3자+ 포함 청크 | "4.5%" 보고 | **35.0%** (대부분 1~3자 PUA 잔재) |
| "공인결석 매뉴얼 77.8%" | 잘못된 측정 | 정상 (PUA 1~2자만 있음) |
| "학사안내 PDF 7.9%" | 잘못된 측정 | 정상 수준 |
| "HWP 학사안내 4.9%" | 잘못된 측정 | 정상 수준 |

**여전히 깨짐 비율 높은 source** (정정 후):
- `2026년 1학기 선원가족 장학생 선발요강.pdf` — 글자 19.5%
- 일부 `.xlsx` 파일들 — 5~6%

→ 대부분 PDF·HWP는 거의 정상. **이전 보고서의 깨짐 관련 결함 #5(중복) 외 깨짐 관련 진단은 무효**.

## 6. 개선 권고 (우선순위 순)

### P0 — 긴급 (Week 1)

1. **이중 색인 제거** (#1) — 동일 내용 PDF/HWP/크롤판 중복 검출 후 단일화
   - 방법: 첫 1KB SHA256 해시 기반 dedup. 이미 구현된 `cleanup_duplicate_chunks.py` 확장.
   - 기대 효과: 인덱스 크기 30~40% 감소, 검색 노이즈 감소

2. **`student_types` 추출 로직 복원** (#2) — 청킹 시 실제 값 채우기
   - 방법: 문서명·섹션 헤더에서 "내국인/외국인/편입생" 추출. 문서 루트에서 기본값 지정.
   - 기대 효과: 학생유형별 필터 작동

3. **`detect_cohort()` regex 강화** (#3) — "이후/이전/학년도부터" 패턴 추가
   - 방법: 학번 추출 휴리스틱 확장 + 문서 파일명/제목에서 기본 연도 폴백.
   - 기대 효과: 학번 특화 청크 비율 0.8% → 20%+ 기대

4. **`department` 메타 주입 전체 문서로 확장** (#4) — 시간표 외 청킹에도 적용
   - 방법: 학사안내 PDF의 학과 목록 섹션·공지의 학과명 패턴 추출
   - 기대 효과: 학과별 필터 실작동

### P1 — 중요 (Week 2)

5. **중복 청크 21.8% 정리** (#5) — 크롤 본문 추출 정제
   - 방법: `<nav>`, `<aside>` 태그 명시적 제외. 마크다운 포털 테이블 패턴 필터.
   - 기대 효과: 네비게이션 노이즈 제거

6. **`section` 메타 주입 복원** (#6) — 섹션 경로 저장 방식 재설계
   - 방법: flat 저장(`section: "3.2 졸업요건"` 단일 문자열) 또는 JSON 직렬화
   - 기대 효과: 출처 표기 계층화, 섹션별 필터 가능

7. **`category` 메타 전 문서 확장** (#7) — FAQ 외 카테고리 분류 주입

### P2 — 정제 (Week 3)

8. **청크 길이 hard cap** (#8) — sliding window 후 500자 초과 강제 분할
9. **부재 카테고리 FAQ 보강** (#9) — 개명·학생증 등 6~16개 청크만 있는 항목 확장
10. **FAQ 노드 direct_answer 저장 구조 검증** (#10) — 노드 속성 vs 엣지 참조 방식 결정

## 9. 공유 대상별 요약

### 기술팀 / 개발자
- **핵심 메시지**: 인덱스 데이터는 있으나 메타가 망가짐. 청킹 단계 추출 로직 전면 재검 + 크롤 dedup 필요.
- **즉시 확인 명령**:
  ```bash
  python3 scripts/data_fingerprint.py  # 상세 통계
  sqlite3 data/chromadb_new/chroma.sqlite3 \
    "SELECT key, COUNT(*) FROM embedding_metadata GROUP BY key"
  ```

### PM / 팀원
- **핵심 메시지**: 최근 여러 개선 커밋이 "코드는 있는데 데이터는 없는" 상태라 효과가 낮게 나옴. 2~3주 재색인 중심 개선이 필요.
- **결과 체감**: 학번별 맞춤답변·학과별 필터링 등 홍보 기능이 실제로는 95% 무작동 상태였음.

### 학사지원팀
- **핵심 메시지**: OPS 11 유형 **데이터는 모두 갖춰져 있으나** 개명(6청크)·학생증(16청크) 등 일부 항목은 FAQ 추가 제공이 답변 품질 향상에 도움됨.
- **요청**: OCU·특별강좌·학생증 3종 구분·개명 절차 관련 문안 제공 (각 2~3개)
